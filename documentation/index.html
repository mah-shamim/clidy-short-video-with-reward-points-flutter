Admin Panel Configuration

1. Upload Files to Server

Upload the admin panel files to your server using cPanel. Ensure each file is uploaded correctly, and verify that the directory structure matches the original project.

Note: If you encounter issues while uploading files, check folder permissions and make sure all essential files, like index.php and configuration directories, are present.

2. Create Database

Create a database in cPanel with a specific name and user for your application. Then, use the database import tool to load the .sql file included in the admin panel project.

Tip: When importing the .sql file, ensure the encoding is compatible (UTF-8 is usually recommended). This will prevent character errors in the database.

3. Configure Database Connection

Edit the database.php file to set up database connection details. You will need to provide the database name, user, and password you configured in the previous step.

Note: Ensure connection details are correct, and that the database is on the same server or that the database server allows remote connections.

App Configuration

1. Open Project

Open the Clidy project in Android Studio or Visual Studio Code. Ensure all necessary Flutter dependencies are installed before proceeding.

Tip: If you encounter issues opening the project in Android Studio, try synchronizing dependencies and verifying that the Flutter SDK is correctly configured.

2. Run pub get to Compile Dependencies

In the pubspec.yaml file, run the pub get command to compile dependencies. This will download and install all necessary libraries for the project.

Tip: If dependencies do not install correctly, check your internet connection and ensure the Flutter version is compatible with the project.

3. Change App Name

To customize the app name, edit the following files:

  • AndroidManifest.xml - Change the name that appears to the user on their device.
  • pubspec.yaml - Update the name to reflect your app's branding.

4. Change Package Name

Configure the package name to customize the application identifier in the build.gradle and AndroidManifest.xml files.

  • build.gradle - Update the applicationId to the new desired package name.
  • AndroidManifest.xml - Change the package attribute to the new package name.

Note: The package name must be unique to avoid conflicts on the Play Store.

5. Configure Admin Panel Base URL

Open the api.dart file and set up the base URL for your admin panel. This will allow the app to communicate with the server correctly.

Tip: Ensure the URL is accessible from the device on which you are testing the application.

6. Generate Keystore

To sign the app, generate a keystore using the following command in the terminal. This is required to upload the app to the Play Store.

keytool -genkey -v -keystore .keystore -keyalg RSA -keysize 2048 -validity 10000 -alias

Tip: When generating the keystore, be sure to remember the password and alias, as they will be needed for future app updates.

Generate Web Files with Flutter

7. Generate Web Files with Flutter

To generate your Flutter app files in web format, follow these steps:

  1. Run the following command in your terminal:
    2. flutter build web

    This command will generate the files needed to run your application as a web app. The resulting files will be located in the build/web folder of your project.

  2. Enable the Flutter channel for web if you haven't already:
    3. flutter config --enable-web

    Ensure the Flutter channel is enabled for web to generate the files correctly.

  3. Upload all generated files to your domain or subdomain:

    4. After generating the files, upload the content from the build/web folder to the root of your domain or subdomain to publish your web application.
